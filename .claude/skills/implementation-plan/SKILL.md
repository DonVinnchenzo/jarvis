# Implementation Plan

## When to Use
After a spec is approved, before writing any code.

## Steps

1. Read the approved spec
2. Map all files that need to be created or modified
3. Identify dependencies between changes (what must come first)
4. Break into implementation steps, each resulting in one commit:
   - Step N: What to do, which files, what to test
5. Identify risks and edge cases
6. Save plan to `Ideation/[FEATURE]-IMPLEMENTATION-PLAN.md`
7. Present plan to user for approval

## Validation
Done when plan is saved and user approves.

## References
- Approved spec in `specs/`
